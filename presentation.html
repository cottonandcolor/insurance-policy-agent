Problem & Intended User

  • 30–80 page policy documents written in dense legal language
  • Comparison sites rank by premium, not coverage at claim time
  • Example: flood zone — Plan A excludes flood; Plan B covers with $100K sublimit
  • Users: consumers comparing household coverage; brokers evaluating client options

System Goal & Scope

  • Input: user profile + insurance plan documents
  • Output: cited comparison table + ranked recommendation
  • Scope: advisory decision support — no binding quotes or purchases
  • Data: synthetic/public/anonymized policies only

Six Agent Roles

# Role Responsibility LangGraph node(s) Output
1 Orchestrator Phase routing, ToT loop control, beam/depth limits, LLM budget (≤25 calls) tot_init · advance_depth · conditional edges Workflow state, loop exit
2 Profile Intake Parse user requirements — location, flood zone, home value, priority weights intake session_profile
3 Document & Retrieval Chunk policies by section, index in Chroma, normalize plans, attach evidence at query time index · ingest · ground normalized_plans · chunk IDs
4 Scenario Reasoning Generate ToT branches — e.g. flood excluded vs sublimited, $150K loss scenarios tot_expand active_branches + thoughts
5 Policy Critic Adversarial scoring — grounding, consistency, payout validity; recommends prune evaluate (+ hard_gate validators) composite_score · prune flag
6 Recommendation Synthesizer Build cited comparison table and advisory recommendation from winning branch synthesize final_recommendation

Why separate roles?

  • Reasoning and criticism use different prompts — reduces self-confirmation bias
  • Retrieval and validation run between reasoning steps — citations required before scoring
  • Hybrid coordination: sequential intake → iterative ToT graph → sequential synthesis

Key Design Decisions

DecisionWhy
Retrieval requiredPrevent hallucinated coverage claims
Multi-agent (6 roles)Separate reasoning from criticism
ToT + beam search (width 3)Handle interpretive ambiguity
Fail-closed safetyEscalate to human when evidence is missing

End-to-End Run — OpenRouter + Client → Server Flow

Setup & run (all steps)

1. git clone …/insurance-policy-agent && cd capstone
2. python -m venv .venv && source .venv/bin/activate
3. pip install -r requirements.txt
4. cp .env.example .env  →  configure OpenRouter:

   LLM_PROVIDER=openai
   OPENAI_API_KEY=sk-or-your-openrouter-key
   OPENAI_BASE_URL=https://openrouter.ai/api/v1
   OPENAI_MODEL=meta-llama/llama-3.3-70b-instruct:free

5. Terminal 1 (project root):
   uvicorn api.main:app --reload --port 8000

6. Terminal 2:
   cd frontend && npm install && npm run dev

7. Browser: http://localhost:5173
   · Uncheck Dry-run · Check Quick mode
   · Choose policies → Analyze Plans

Safety & Human Oversight

  • Guardrails: citations required · no purchase APIs · LLM budget · separate Critic
  • Escalate when: evidence gap · low confidence · all branches pruned · >$25K payout divergence
  • Metrics: ≥95% grounding · ≥90% payout accuracy · 100% fallback on hard-gate failure
Input → Retrieval → ToT + Critic → Hard gates → Output OR Human handoff
