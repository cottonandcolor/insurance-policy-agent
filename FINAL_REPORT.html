1. Problem Statement and Intended User

Real-world problem

Choosing insurance is one of the most consequential financial decisions a household makes, yet it remains opaque. Consumers face dozens of plans with different limits, deductibles, exclusions, and riders buried in 30–80 page legal documents. Comparison sites rank by premium, not by what happens at claim time. A flood-zone consumer may choose a cheaper plan that excludes flood entirely — exposing the household to catastrophic loss.

Intended user

  • Primary: Individual consumers comparing plans for their household
  • Secondary: Insurance brokers evaluating options for a client

2. System Goal, Scope, and Constraints

Goal

Autonomously collect user requirements, ingest and normalize policies, retrieve grounded evidence, explore interpretive reasoning paths (Tree-of-Thought), score and prune weak hypotheses, and recommend the best-fit plan with cited tradeoffs.

Scope

In scope: Auto/home/life comparison, loss-scenario analysis, cited comparison tables, follow-up re-scoring, synthetic/public data only.

Out of scope: Binding quotes, policy purchase, claims filing, proprietary client data, medical underwriting.

Constraints

  • Synthetic/public/anonymized policy data only
  • LLM budget: ≤25 calls, ≤45s per cycle
  • Beam search: width 3, depth 4
  • Advisory only — human review under defined escalation criteria

3. Final System Architecture

The system integrates RAG, ToT beam search, multi-agent role separation, and safety guardrails — orchestrated by LangGraph with CrewAI agents.

Phase 1   Profile Intake Agent (CrewAI)
Phase 2   Chroma Index + Section-Aware Chunking
Phase 3   Document & Retrieval Agent → Normalized Schema
Phase 4–5 ToT Loop: Reasoning → Ground → Hard Gates → Critic → Prune/Expand
Phase 6   Recommendation Synthesizer Agent
Shared State: session_profile, branch_store, retrieval_cache

LangGraph routes phases and runs the ToT beam loop. CrewAI agents handle specialized reasoning. Chroma provides grounded retrieval. Hard gates block uncited claims before the Policy Critic scores surviving branches. The Synthesizer produces the final cited recommendation from the winning branch only.

8. External APIs and Session Memory

External APIs (enrichment layer)

The agent integrates public APIs to enrich the user profile without replacing policy-grounded reasoning. Coverage claims still require uploaded policy text and Chroma retrieval.

APIPurposeIntegration
US Census GeocoderConvert location text to lat/lonenrich node after intake (src/tools/location_enrichment.py)
FEMA NFHL ArcGIS RESTOfficial flood zone code (AE, X, etc.) for coordinatesSame enrichment node; stores flood_zone_code in session_profile
OpenFEMA (optional)NFIP community/program contextFuture work — cited in design for broker-facing context

Enrichment is fail-open: if an API is unreachable, the workflow continues using form inputs. When NFHL responds, the UI displays the inferred zone on the results panel.

Session memory (multi-turn interactivity)

Interactivity is implemented with structured state, not raw chat history alone:

  • LangGraph checkpointer (MemorySaver) — thread-scoped graph state via thread_id
  • Session store (data/sessions/) — persists normalized_plans, retrieval_cache, conversation_history, and run settings between API calls
  • Follow-up routing — POST /api/follow-up re-enters the workflow at tot_init with start_depth=2, skipping re-index and re-ingest when plans are cached
  • Conversation trim — last 8 turns kept to avoid context overflow; assistant summaries stored after synthesis
First analyze:  intake → enrich → index → ingest → ToT loop → synthesize  →  thread_id
Follow-up:      intake → enrich → tot_init → ToT loop → synthesize  (reuse normalized_plans)

9. Safety, Reliability, and Human Oversight

Guardrails: Synthetic data only; citation-required grounding; arithmetic validator; no purchase APIs; LLM budget; separate Critic; output disclaimer; block output if all branches fail.

Human intervention when: Evidence gap after expanded retrieval; low confidence with tied scores; all branches pruned; payout divergence >$25K; user corrects misread term; out-of-scope request (binding quote, purchase, claims).

Input validation → Grounded retrieval → ToT + Critic → Hard gates → Metrics check → Output OR escalate

10. Conclusion

This capstone delivers an autonomous insurance comparison agent that integrates retrieval, multi-agent collaboration, Tree-of-Thought reasoning, and safety guardrails into a cohesive LangGraph + CrewAI implementation — addressing a real consumer and broker need with testable, auditable controls.
